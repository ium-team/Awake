# macOS 권한 정책

## 기본 원칙

MVP는 가능한 한 권한 없이 동작한다. 권한이 필요한 기능은 사용자가 그 기능을 쓰려는 시점에 요청하거나 안내한다.

## 권한별 사용

### Accessibility

필요할 수 있는 경우:

- 앱별 상세 윈도우 제목 확인
- 특정 윈도우 상태 추적
- 전역 단축키 구현 방식에 따라 필요한 경우

MVP에서는 Accessibility 권한이 없어도 앱 단위 보호 세션은 가능해야 한다.

### Notifications

자동 종료 알림을 보낼 때 필요하다. 알림 권한이 없어도 세션 기능은 동작해야 한다.

### Login Items

MVP 범위 밖이다. 후속 버전에서 로그인 시 실행 옵션을 추가할 때 다룬다.

## 권한 UX

- 권한이 없으면 기능 전체를 막지 않는다.
- 권한이 필요한 정보만 제한한다.
- 설정 앱으로 이동할 수 있는 버튼을 제공한다.
- 권한 허용 후 목록 새로고침을 제공한다.

## 실패 메시지 원칙

- 사용자가 무엇을 허용해야 하는지 명확히 쓴다.
- 권한 없이도 가능한 대안을 보여준다.
- macOS 버전이나 관리 장비 정책 때문에 막힐 수 있음을 숨기지 않는다.
