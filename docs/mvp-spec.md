# MVP 명세서

## 목표

Awake MVP는 사용자가 선택한 실행 중 앱/윈도우가 종료될 때까지 macOS 유휴 잠자기를 막는 메뉴바 앱을 만든다. 첫 버전은 신뢰 가능한 전원 assertion, 명확한 상태 표시, 자동 해제에 집중한다.

## 사용자 흐름

### 보호 세션 시작

1. 사용자가 메뉴바 아이콘을 클릭하거나 전역 단축키를 누른다.
2. Awake가 현재 실행 중인 앱과 감지 가능한 윈도우 목록을 표시한다.
3. 사용자가 하나 이상의 대상을 선택한다.
4. 사용자가 `Start`를 누른다.
5. Awake가 전원 assertion을 생성하고 세션을 active 상태로 바꾼다.
6. 메뉴바에 active 상태와 선택 대상 수를 표시한다.

### 보호 세션 유지

- Awake는 선택 대상의 프로세스 생존 여부를 주기적으로 확인한다.
- 선택 대상 중 하나라도 살아 있으면 세션은 유지된다.
- 선택 대상이 모두 종료되면 세션을 자동 종료한다.
- 사용자는 메뉴바에서 언제든 수동 종료할 수 있다.

### 보호 세션 종료

- 자동 종료 조건: 선택한 대상이 모두 종료됨
- 수동 종료 조건: 사용자가 `Stop Keeping Awake` 선택
- 앱 종료 조건: Awake 종료 시 assertion을 반드시 해제

## 범위

### 포함

- macOS 메뉴바 앱
- 실행 중 앱 목록 표시
- 가능한 경우 앱별 윈도우 목록 표시
- 다중 선택
- 전원 assertion 생성/해제
- 선택 대상 종료 감지
- 세션 상태 메뉴바 표시
- 권한 부족 안내
- 기본 설정 저장

### 제외

- private API 사용
- 덮개 닫힘 계속 실행 보장
- 작업 내용 분석
- 터미널/tmux 세션 내부 프로세스 추적
- 앱별 자동 추천
- iCloud 동기화
- 로그인 아이템 자동 등록

## 상태 모델

- `idle`: 보호 세션 없음
- `selecting`: 대상 선택 화면 열림
- `active`: assertion이 켜져 있고 대상 감시 중
- `stopping`: assertion 해제 중
- `error`: assertion 생성 실패, 권한 부족, 감시 실패 등 사용자 조치 필요

## 세션 종료 규칙

MVP에서는 선택한 대상 중 하나라도 살아 있으면 보호 세션을 유지한다. 모든 대상이 종료되면 해제한다.

윈도우 단위 선택은 UI에서 제공하되, 실제 종료 감지는 앱 프로세스 단위로 시작한다. 윈도우 단위 자동 종료는 macOS 앱별 윈도우 동작 차이가 크므로 후속 버전에서 정교화한다.

## 사용자 메시지 원칙

- "덮개를 닫아도 항상 됩니다"라고 쓰지 않는다.
- "일반 잠자기를 막고, 환경에 따라 덮개 닫힘 동작은 달라질 수 있습니다"라고 안내한다.
- 실패하면 사용자가 다음에 할 행동을 알 수 있게 쓴다.

## 기본 설정

- 기본 assertion 모드: system sleep 방지 + display sleep 허용
- 체크 간격: 5초
- 세션 종료 후 알림: 켬
- 메뉴바 active 표시: 켬
